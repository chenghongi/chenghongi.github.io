对象中成员的访问

对象中包含成员属性和成员方法,可以通过对象操作符 -> 来访问对象的属性或方法,
静态属性则使用 双冒号 :: 来访问.
在类成员方法的内部调用属性时要使用$this 这个伪变量. $this只代表当前对象.

1. $引用名 = new 类名称([参数列表]); //实例化对象
2. $引用名->成员属性 = 值;  //给成员属性赋值
3. $引用名->成员方法();  //访问成员方法
普通对象访问类中成员属性或方法: "对象引用名->属性" 或 '对象引用名->方法' 的形式进行访问 不管是在对象的外部还是在对象的内部访问对象中的成员都必须使用对象的引用变量名来访问. 在类中使用 `$this` 代表当前对象.加上特殊操作符号 `'->'`; 静态属性或方法是属于类的,不属于对象,在多个对象之间共享的,类的静态属性非常类似于函数的全局变量,用类名直接访问, enter code here 类名:: 静态成员属性; 类名::静态成员方法(); 在类中声明的成员方法中,只用关键字 `"self"` 来方法静态成员,因为静态成员是属于类的,所以不能用 对象的 `$this` . 关键字访问, 而`"self"` 关键字就是在类的成员方法中用来代表本类的关键字 enter code here self:: 静态成员属性 self:: 静态成员方法(); 在静态方法中只能访问静态成员, (因为非静态的成员必须通过实例化的对象引用才能访问) 但是可以使用类名或`self(推荐self)`在非静态方法中访问静态成员. #### 构造方法和析构方法 >这俩个特殊的方法都与对象的生命周期有关,构造方法是对象创建是第一个自动被调用的方法,而析构方法是在对象销毁之前被自动调用的一个方法. ##### 构造方法`__construct()` 每当用关键字 `new` 实例化创建一个对象时,构造方法就会被自动调用一次,我们可以用构造方法的这个特性执行一些初始化任务,比如赋值. 每个类中只能有一个构造方法(默认有一个不带任何参数列表并且内容为空的构造方法) ##### 析构方法 `__destruct()` 析构方法的声明和构造方法类似,都是固定的方法名,但析构方法不能带有任何参数,它属于类中可选的一部分,不是很常用. #### 类的三大特性: 封装 - 继承 - 多态 ##### 封装 　　将数据和操作封装为一个有机的整体，由于类中私有成员都是隐藏的，只向外部提供有限的接口，所以能够保证内部的高内聚性和与外部的低耦合性。用者不必了解具体的实现细节，而只是要通过外部接口，以特定的访问权限来使用类的成员，能够增强安全性和简化编程。 ##### 继承 继承性更符合认知规律，使程序更易于理解，同时节省不必要的重复代码。 ##### 多态 同一操作作用于不同对象，可以有不同的解释，产生不同的执行结果。在运行时，可以通过指向基类的指针，来调用实现派生类中的方法。 #### 访问控制关键字 1. public 公开的 2. protected 受保护的, 3. private 私有的 ,只能在对象内部的成员方法中使用,不能被对象外部直接赋值, 默认public , 公开的,外部可以访问. #### 预定义方法, 预定义方法 : 每个方法都有特定的作用,不需要直接调用,而是在特定的条件下自动被调用,都是以俩个下划线开始的方法名称,也称作魔术方法.这些魔术方法只有__construct构造方法类中默认存在,其他都得根据需要自行添加. 
>__set() ; __get(); __isset()和__unset()4个方法,包括前面的__construct()
以及__destruct();
